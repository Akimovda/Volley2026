**9 из 10 строк в топе — пустые/нулевые**, ровно как в описанном симптоме («бар только у ОДНОГО из 4+»). Причина — PostgreSQL `ORDER BY ... DESC` кладёт `NULL` первыми, `LIMIT 10` в первую очередь набрал события без строк в `event_occurrence_stats`.

**Дополнительная находка**: даже единственная «живая» строка (event 364: 140/97.2%) была искажена — старый запрос усреднял `avg_load_pct` только по occurrences, у которых СЛУЧАЙНО была строка в кеше `event_occurrence_stats` (обычно это occurrences с заметной регистрацией — кеш чаще создаётся/обновляется при реальных действиях). Occurrences с низкой/нулевой явкой, у которых кеш не создавался, молча исключались из среднего — получалась смещённая (survivor bias) оценка 97.2% вместо реальной.
