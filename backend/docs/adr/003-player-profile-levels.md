# ADR-003: Player Profile Levels, Lock, Roles

## Статус
Принято

## Контекст
Нужно:
- хранить уровни игрока в базе и сравнивать их с требованиями мероприятий
- не показывать “цифры” пользователю в профиле
- обеспечить принцип “уровень выбирается один раз” (самоопределение)
- ограничить уровни для игроков младше 18 лет
- разрешить корректировку уровня администраторам/организаторам

## Решение
1) Уровни храним в БД как числа (1..7), UI отображает названия через конфиг.
2) Для <18 разрешаем только уровни: 1, 2, 4 (Начальный, Начальный+, Средний).
3) Пользователь может выбрать уровень только один раз:
   - если уровень уже задан и пользователь не имеет прав — изменение не принимается.
4) Корректировать уровень могут роли: admin/organizer/assistant через Gate `edit-player-levels`.

## Причины
- сравнение требований мероприятий по числам проще и надежнее
- UX: пользователю понятнее названия, а не цифры
- защита от “самоприсвоения” высоких уровней
- оргкоманде нужна возможность ручной корректировки

## Последствия
- требуется `birth_date` в профиле
- требуется механизм ролей/прав (минимально: поле role и Gate)
- требуется страница определения уровней `/levels` для самоопределения
