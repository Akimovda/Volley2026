# ADR-001: Authentication Strategy

## Статус
Принято и реализовано

## Контекст
Проект Volley требует гибкой авторизации с возможностью
объединения аккаунтов и масштабирования.

## Решение
- Laravel Jetstream + Fortify
- Telegram Login как primary identity
- VK ID как secondary provider
- Один пользователь — несколько способов входа
- Проверка заполненности профиля вынесена из логина
- Контекстная проверка при записи на мероприятие

## Причины
- Telegram — основной канал коммуникации проекта
- VK широко используется в РФ
- UX: не блокировать пользователя при входе
- WebAuthn (Passkey) — будущее без паролей

## Последствия
- Усложнение логики профиля
- Необходимость сервисов требований (`EventRegistrationRequirements`)
- Чёткое разделение:
  - Login → только аутентификация
  - Action → проверки данных
- Повышенная безопасность и управляемость UX
