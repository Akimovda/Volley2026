# Roadmap — Volley

## Текущая версия
**v0.1-alpha**

Проект находится на стадии:
- завершён базовый пользовательский поток
- архитектура стабилизирована
- UX-паттерны зафиксированы
- документация актуализирована

Дальнейшее развитие идёт **итеративно**, без слома существующих решений.

---

## v0.1-alpha (текущая)

### Цель
Зафиксировать рабочий MVP-скелет.

### Реализовано
- Мульти-провайдерная авторизация (Telegram + VK)
- Контекстная проверка профиля
- Анкета игрока
- Автозапись на мероприятия
- Запись / отмена записи
- Страница мероприятий
- Централизованные стили
- Документация и ADR

### Ограничения
- Нет админки
- Нет команд
- Нет лимитов участников
- Нет верификации телефона

---

## v0.2 — Управление мероприятиями (Admin)

### Цель
Дать возможность **создавать и управлять мероприятиями** без вмешательства в БД.

### Планируется
- Admin UI:
  - CRUD мероприятий
  - управление требованиями
- Признак организатора и помощника организатора
- Скрытые/черновые мероприятия
- Лимит участников (soft-limit)
- Просмотр списка участников

### Архитектурно
- Отдельный namespace: `Admin`
- Использование существующих сервисов требований
- Без изменения UX игрока

---

## v0.3 — Команды и распределение игроков

### Цель
Перейти от “списка участников” к **структурированной игре**.

### Планируется
- Команды внутри мероприятия
- Ручное и автоматическое распределение
- Учет уровней игроков
- Простые роли:
  - игрок
  - капитан
  -помошник организатора
  - организатор

### UX
- Игрок видит свою команду
- Организатор и помощник управляет составами

---

## v0.4 — Профиль и Merge UI

### Цель
Закрыть долг по качеству данных и поддержке реальных пользователей.

### Планируется
- UI объединения аккаунтов (merge)
- Подтверждение телефона через Telegram
- История изменений профиля
- Улучшение анкеты игрока

### Безопасность
- Merge только явно
- Подтверждение критичных действий

---

## v0.5 — Уведомления и коммуникация

### Цель
Сделать Volley живым сервисом, а не просто страницей.

### Планируется
- Telegram-уведомления:
  - запись на мероприятие
  - изменения состава
  - напоминания
- или VK Уведомления (опционально):
- запись на мероприятие
  - изменения состава
  - напоминания
- Email (опционально)
- Центр уведомлений (in-app)

---

## v1.0 — Production Ready

### Цель
Стабильный продукт для реального использования.

### Планируется
- Passkey (WebAuthn)
- Полная role-based access model
- Audit log
- Rate limiting
- Monitoring
- Backup & recovery

### Критерии готовности
- Нет блокирующих UX-флоу
- Все критичные действия задокументированы
- Тесты на ключевые сценарии
- Обратная совместимость

---

## Принципы развития

- ❌ Не ломать существующие флоу
- ❌ Не проверять профиль при логине
- ✅ Проверки только по действию
- ✅ UX важнее формальной строгости
- ✅ Документация обновляется вместе с кодом
